# Domain: shared (newb ↔ allow 공유 모델)

두 사이트 모두에 공통으로 적용되는 도메인 정책.

## 1. 친구 관계 깊이 — "중간"

사용자가 명시한 정합 수준:

| 항목 | 적용 |
|------|------|
| 사이트 스위처 / footer 링크 | ✅ 둘 다 노출 (양방향 상호이동) |
| 회원 / 세션 공유 | ✅ 같은 ID = 양쪽 자동 회원 |
| 한쪽 메인에 다른쪽 최신 카드 1줄 | ✅ (newb 메인에 "허락 최근 방송/콘테스트", allow 메인에 "뉴비 훈련소 공지" 1줄) |
| 통합 피드 / 통합 글 DB | ❌ 글은 사이트별 분리 — 같은 회원이 양쪽에 쓰지만 sync 는 안 함 |
| 활동 sync (한쪽 글이 다른쪽 프로필에 표시) | ❌ 깊이 과함. 백로그도 아님. |

**원칙**: 회원·인증·자료(R2) 는 통합. 콘텐츠·운영·UI 톤은 사이트별 독립.

## 2. 회원·권한 모델

**회원 분리하지 않음.** 한 번 가입 = `dnfm.kr` / `www.dnfm.kr` / `allow.dnfm.kr` 자동 회원.

- 인증: `api.dnfm.kr` (Stage 2). JWT or 세션 쿠키, 쿠키 도메인 `.dnfm.kr` 로 발급해서 sibling subdomain 공유.
- 권한 (역할) 은 **사이트별 분리**:
  - newb 어드민 ≠ allow 어드민. 같은 회원이라도 권한은 사이트마다 별도 부여.
  - 권한 enum 예 (확정 X, 백로그): `member`, `newb_admin`, `allow_admin`, `super_admin`.
  - super_admin = 사용자(방장) 본인. 양쪽 다 관리 가능.
  - allow_admin = 허락님. allow 콘테스트/투표/공지 self-service ([[allow]] §2).
- 비회원: 사이트 열람은 가능, 글쓰기/투표/콘테스트 참가는 회원만.

## 3. 사용자 업로드 (R2)

- 사용자 업로드(allow 의 코디 사진, 향후 newb 의 이벤트 자료 등) 는 모두 R2 한 버킷.
- 발급 경로: 프론트 → `api.dnfm.kr` 의 presigned PUT 요청 → 직접 R2 업로드 → 완료 후 메타데이터를 백엔드에 confirm.
- R2 public 차단. 다운로드도 백엔드 presigned GET 또는 worker route 중계.
- 사용자가 만든 자료 (게시글 본문, 업로드 이미지) 는 [[anti-avoidance §8]] / [[code-quality]] 정책 따라 **자동 변경 영구 금지**. 어드민 삭제는 명시적 UI 통해서만.

## 4. Cross-link 패턴 (중간 깊이 구현 가이드)

**위치별 가시화:**

- **헤더 / 사이트 스위처** — 두 사이트 모두 헤더 우측에 "↔ 자매 사이트" 1버튼 (newb → 허락 페이지, allow → 뉴비 훈련소). 시각 톤은 사이트별 자율.
- **footer** — 자매 사이트 링크 + 운영 메모 1줄.
- **메인 1줄 카드**:
  - newb 메인 → "허락님 페이지 — 최근 방송 / 진행중 콘테스트" 카드 1개. allow API 에서 최신 1건 fetch.
  - allow 메인 → "뉴비 훈련소 — 오늘의 공지" 카드 1개. newb 자체 정적 또는 api 에서 최신.
- **회원 페이지** — "내 활동" 같은 통합 페이지는 만들지 않음. 사이트별 본인 활동은 사이트 안에서만.

**구현 시 주의:**
- 한쪽 다운 시 다른쪽도 카드 영역만 fallback 으로 (네트워크 분리 X — 같은 백엔드라 SPOF).
- cross-link 카드 1개당 1 API call 캡. 페이지 로딩 자체 느려지지 않게 lazy.

## 5. 디자인 톤 분리 원칙

- 같은 코드베이스 / 같은 컴포넌트 라이브러리 (있다면) / 같은 디자인 토큰 SSOT.
- 그러나 **사이트별 theme** 으로 시각 분리:
  - newb: 라이트 / 공식 게시판 톤 (자체 브랜드 추상화 후)
  - allow: B급 감성 / 비대칭 / 위트 ([[allow]] §3)
- 컴포넌트 새로 만들 때 — 두 사이트 모두에서 쓸 수 있게 theme prop 또는 CSS variable 로 분기.

## 6. 트레이드마크 / 공식 콘텐츠 회피

- 두 사이트 모두 NEXON/NEOPLE 의 공식 wordmark/로고/공식 게시판 카테고리/공식 UI 톤을 directly copy 하지 않음.
- 공식 콘텐츠로 연결되는 외부 링크는 OK. 자체 페이지에 공식 톤·표기 모방 X.
- footer disclaimer 표준: `"던전앤파이터 모바일은 NEXON/NEOPLE의 게임입니다. 이 사이트는 비공식 [커뮤니티/팬] 페이지입니다."`

## 7. 향후 친구 사이트 확장

사용자 발언: "다른 친구에게도 페이지 제공 가능성." 그 때 패턴:
- 새 디렉토리 `<friend>/` 추가, 호스트 `<friend>.dnfm.kr`.
- 같은 회원/인증/R2/api 공유. 권한 enum 에 `<friend>_admin` 추가.
- shared.md cross-link 정책에 따라 메인 카드 1줄 상호 노출.
- 사이트별 톤은 친구가 정함 (B급/공식/미니멀 등).
